# ADR 001: DB Outbox를 선택한 이유

## 결정

주문 후속 처리는 Kafka 같은 외부 브로커 대신 PostgreSQL 기반 outbox 테이블로 구현한다.

## 이유

- 포트폴리오 저장소 하나로 재현 가능한 데모를 만드는 것이 우선이다.
- 메시지 브로커 없이도 비동기 후속 처리 패턴을 설명할 수 있다.
- 주문 이벤트를 감사 가능한 형태로 남길 수 있다.
- portfolio snapshot 같은 후속 처리를 주문 트랜잭션과 느슨하게 분리할 수 있다.

## 결과

- `outbox_event`가 감사 로그와 처리 큐 역할을 동시에 맡는다.
- 같은 서비스 안에서 consumer를 돌려 로컬 데모 복잡도를 낮춘다.
- 향후 메시지 브로커로 이전할 때 aggregate/event 설계는 재사용 가능하다.
